Qual a melhor framework

Escrito por: Mariana Alves

O que é uma framework?

Mas, afinal uma Framework é uma Library. Tem código que podemos reutilizar. Certo?

Vamos lá explicar melhor! Sim são códigos escritos por outra pessoa que ajudam a não "inventarmos a roda" e assim permitem poupar tempo na resolução de problemas comuns. Mas há diferenças. Explora os exemplos:

  • Framework
  • Library

Framework

Tal como construíres uma casa modelo. Já existem plantas que são feitas com base na maioria das necessidades de determinadas famílias. O empreiteiro e o arquitecto controlam e vão dizer o que pode ser feito ou não a nível de alterações.

Library

Imagina que uma Library é como ir ao Ikea. Sim, tem aqueles bolos de canela muito bons, mas vamos nos concentrar no essencial! Tu já tens uma casa, mas precisas de ajuda com algumas peças. Não te apetece fazer um candeeiro do zero. Vais lá e escolhes o que fica melhor na tua casa. Tu controlas.

Resumindo:

Framework tenta adivinhar o que vais precisar, por isso é tão importante saberes se queres usar uma. E aliar essa escolha ao projeto que tens em mãos!

Library é chamada onde e quando precisares.

Devemos usar frameworks?

Convém haver experiência em front-end antes de se utilizar. Porque para se resolver um problema de compatibilidade é preciso esse conhecimento.

Prós
  • Aumenta a eficiência
  • Facilita as repetições de tarefas
  • Arquivos modularizados
  • Solução para os problemas CSS comuns
  • Ajuda no trabalho em grupo
Contras
  • Pode minimizar o conhecimento da linguagem
  • Inflexibilidade na personalização acrescida
  • Geralmente o site fica mais pesado

Para ajudar-te neste processo de escolha vamos analisar 3 frameworks de CSS muito populares:

Bootstrap

Vai na versão 5 que foi lançada a 16 de Junho de 2020

Originalmente desenvolvido para o Twitter, chamado de Twitter Blueprint, desenvolvido por Mark Otto e Jacó Thornton com o objectivo de incentivar a consistência através de ferramentas internas.

Foi renomeado de Bootstrap, e lançado como um projeto de código aberto em 19 de agosto de 2011.

Vê o sistema de grelha de 12 colunas Kitchen Sink todos os elementos da versão 4.2
  • Simplicidade de uso, mesmo para um iniciante;
  • Filosofia de design responsive e mobile-first;
  • Compatibilidade com todos os browsers recentes;
  • Poupança de tempo com componentes reutilizáveis (barras de navegação, dropdowns...);
  • Boa manutenção da plataforma com updates.
  • Classes para esconder e mostrar elementos com facilidade.
  • Classes para acessibilidade, para elementos como leitores de tela e navegação através do teclado.
  • Tamanho grande do ficheiro gerado que pode afetar o tempo de resposta do site;
  • Dependência do HTML e edições necessárias o que pode complicar
  • Todos os sites parecem iguais. Este é o preço a pagar pelo facto de ser fácil de usar. Para diversificar temos que fazer os nossos próprios estilos.

Semantic UI

Vai na versão 2.4.2 que foi lançada a 13 de Novembro de 2016

Desenvolvida por Jack Lukic. Ele é um full-stack developer.

Vê o sistema de grelha de 16 colunas Kitchen Sink todos os elementos
  • De muito fácil utilização;
  • Enorme variedade de temas e personalização;
  • Curva de aprendizagem menor. Demoras menos tempo, já que as classes têm nomes bastante explicativos e intuitivos;
  • Comunidade apaixonada e bastante envolvida.
  • Problema de compatibilidade com IE7;
  • Poucas atualizações
  • Falta de responsividade é causa de preocupação
  • Usa customização por Javascript e isso pode dificultar o trabalho.
  • Packages usadas são muito maiores em tamanho do que as do Bootstrap, por exemplo. E podes só querer usar um módulo específico;
  • Comunidade de apoio ainda em número reduzido;

Materialize

Vai na versão 1.0.0 que foi lançada a 9 de Setembro de 2018

CSS framework que implementa o Material Design da Google.

Foi criada por 4 estudantes na universidade de Carnegie Melon

O foco do Materialize é o visual.

Comportamento físico (movimentos em duas e três dimensões), luz, espaço, etc., para que se obtenha uma uniformidade de reações dos elementos entre as plataformas, ou seja, um botão, por exemplo, tenha o mesmo comportamento em diferentes navegadores e mesmo em diferentes sistemas operacionais, como Windows e Google Android.

Os elementos do website podem ter profundidade e se mover de uma forma específica.

Vê o sistema de grelha de 12 colunas Kitchen Sink de todos os elementos
  • O visual, que permite a criação, com pouco esforço, de websites extremamente interessantes;
  • Redução substancial do tempo de implementação;
  • Documentação cuidadosa e detalhada;
  • Preocupação especial com a experiência do utilizador (UX user experience);
  • Poucos bugs encontrados até o momento, o que reforça a qualidade do projeto;
  • Existência derecursos à medida (cards, flow texts, hoverable items);
  • Grande seleção Plugins em javascript que são gratuitos (Parallax, Modals, Drop-dows...);
  • Grid muito similar à do Bootstrap,mas com um intervalo
  • Complexos componentes Javascript. É difícil mexer quando somos novatos;
  • Não há tanto suporte da comunidade;

Conclusões Finais

A ferramenta correta envolve um grande número de variáveis e depende das especificidades de cada projecto.

O Bootstrap já existe há mais tempo e mantém-se robusta e atualizada. Vence o concurso de popularidade.

Mas existem muitas alternativas interessantes que estão a desenvolver-se, além das que falei anteriormente, também quero destacar a evolução e posição de mercado da Foundation e Tailwind.
